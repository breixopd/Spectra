Getting Started

A quick path from your first target to a defensible security assessment.

API Documentation
01

Define a target

Add an in-scope host, URL, or application from Targets. Keep ownership and authorization details with the assessment record.

02

Run an assessment

Create a mission, choose the approved tools and scope, then monitor progress from the workspace. Stop a run if scope changes.

03

Review evidence

Validate findings against captured evidence, assign owners, and export a report that preserves severity, confidence, and remediation context.

Operate responsibly

Only assess systems you own or have explicit permission to test. Keep scope narrow, document authorization, and treat generated findings as reviewable evidence—not automatic proof.
